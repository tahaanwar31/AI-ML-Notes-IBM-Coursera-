01 — Foundations

What ML is, and what it's made of

AI vs. ML

AI simulates human cognition broadly. ML is narrower: algorithms that learn patterns from data via feature engineering, rather than being explicitly programmed.

Three flavors of learning

Supervised — learns from labeled data.
Unsupervised — finds structure in unlabeled data.
Semi-supervised — trains on a small labeled subset plus a large unlabeled pool.

Four core tasks

Classification — sort into categories.
Regression — predict a number.
Clustering — group without labels.
Anomaly detection — flag what doesn't fit (fraud, defects).

Picking a technique

Comes down to four things: the problem type, the data you actually have, your compute/resource budget, and what outcome you need — interpretable? fast? maximally accurate?

02 — Regression

Linear & logistic regression

Simple Linear Regression

one predictor

y = b0 + b1x

Predicts a continuous target from a single input. Fast and interpretable, but underfits anything nonlinear.

python
from sklearn.linear_model import LinearRegression
model = LinearRegression()
model.fit(X, y)

Polynomial Regression

curved relationships

y = b0 + b1x + b2x² + ...

Use when the relationship clearly bends. Watch for overfitting as the degree climbs.

python
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression

poly = PolynomialFeatures(degree=2)
X_poly = poly.fit_transform(X)
model = LinearRegression().fit(X_poly, y)

Multiple Linear Regression

several predictors

y = b0 + b1x1 + b2x2 + ...

Same idea as simple linear regression, scaled to multiple inputs. Assumes the relationship is still linear.

python
from sklearn.linear_model import LinearRegression
model = LinearRegression()
model.fit(X, y)  # X has multiple columns

Logistic Regression

binary classification

log(p / (1-p)) = b0 + b1x1 + ...

Predicts the probability of a categorical outcome. The workhorse baseline for binary classification.

python
from sklearn.linear_model import LogisticRegression
model = LogisticRegression()
model.fit(X, y)

Picking k for K-Means

Three ways to justify a cluster count instead of guessing it.

Elbow method

Plot inertia vs. k, pick the bend.

inertias = []
for k in range(2, 11):
    km = KMeans(n_clusters=k, random_state=42).fit(X)
    inertias.append(km.inertia_)

Silhouette method

Pick k with the highest score.

scores = []
for k in range(2, 11):
    labels = KMeans(n_clusters=k, random_state=42).fit_predict(X)
    scores.append(silhouette_score(X, labels))

Davies-Bouldin

Pick k with the lowest DBI.

dbis = []
for k in range(2, 11):
    labels = KMeans(n_clusters=k, random_state=42).fit_predict(X)
    dbis.append(davies_bouldin_score(X, labels))

06 — Applied Project

Titanic survival prediction

Logistic regression vs. Random Forest on the classic Titanic dataset — same accuracy, different ideas about what mattered.

Confusion matrix

Visualized true vs. predicted survival outcomes with a heatmap to see exactly where the model was wrong.

Feature coefficients

Pulled logistic regression coefficients, combined numeric + one-hot encoded feature names, and ranked by magnitude.

The actual finding

Logistic Regression and Random Forest scored almost identically — but flagged different features as important. That's a signal of correlated variables (e.g. who_man / who_woman / who_child), not a contradiction. Feature importance needs correlation analysis before you trust it.

python — confusion matrix & feature importance
# Confusion matrix
conf_matrix = confusion_matrix(y_test, y_pred)
plt.figure()
sns.heatmap(conf_matrix, annot=True, cmap='Blues', fmt='d')
plt.title('Titanic Classification Confusion Matrix')
plt.xlabel('Predicted'); plt.ylabel('Actual')
plt.tight_layout(); plt.show()

# Feature coefficient magnitudes
coefficients = model.best_estimator_.named_steps['classifier'].coef_[0]
feature_names = numerical_features + list(
    model.best_estimator_.named_steps['preprocessor']
    .named_transformers_['cat'].named_steps['onehot']
    .get_feature_names_out(categorical_features)
)

importance_df = pd.DataFrame({'Feature': feature_names, 'Coefficient': coefficients}) \
    .sort_values(by='Coefficient', ascending=False, key=abs)

plt.figure(figsize=(10, 6))
plt.barh(importance_df['Feature'], importance_df['Coefficient'].abs(), color='skyblue')
plt.gca().invert_yaxis()
plt.title('Feature Coefficient Magnitudes — Logistic Regression')
plt.xlabel('Coefficient Magnitude')
plt.show()

test_score = model.best_estimator_.score(X_test, y_test)
print(f"Test set accuracy: {test_score:.2%}")
